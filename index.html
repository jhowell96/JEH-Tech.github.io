How it works

  1. Push to main
  2. GitHub Actions uploads the repository as an artifact
  3. actions/deploy-pages publishes it

Your workflow uploads path: '.', so Pages serves whatever is in the repo root — including this index.html.

Customize

  • Edit index.html content
  • Tweak styling in styles.css
  • Add simple interactivity in script.js

Want a blog/docs later? You can keep this workflow and add a build step (e.g., Vite/Next static export), then upload the build output instead of '.'.
